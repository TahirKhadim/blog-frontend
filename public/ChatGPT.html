If you prefer not to use axios, you can use the built-in fetch API to handle HTTP requests. The fetch API is a native JavaScript feature that can handle network requests with a simpler syntax and does not require additional libraries.

Here’s how you can replace axios with fetch in your form submission:

Updated Code Using fetch

jsx
import { useState } from 'react';

export default function App() {
  const [input, setInput] = useState({
    name: '',
    email: '',
    msg: '',
  });

  const handleChange = (e) => {
    const { name, value } = e.target;
    setInput((prev) => ({
      ...prev,
      [name]: value,
    }));
  };

  const onSubmit = async (e) => {
    e.preventDefault();
    console.log(input);
    try {
      const response = await fetch('https://www.greatfrontend.com/api/questions/contact-form', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
        },
        body: JSON.stringify(input),
      });

      if (!response.ok) {
        throw new Error('Network response was not ok');
      }

      const result = await response.json();
      console.log('Success:', result);
      // Handle success, e.g., show a message or reset the form
    } catch (error) {
      console.error('Error submitting form:', error);
      // Handle error, e.g., show an error message
    }
  };

  return (
    <form onSubmit={onSubmit}>
      <label>Name:</label>
      <input
        type="text"
        name="name"
        value={input.name}
        onChange={handleChange}
      />
      <br />
      <br />
      <label>Email:</label>
      <input
        type="email"
        name="email"
        value={input.email}
        onChange={handleChange}
      />
      <br />
      <br />
      <label>Message:</label>
      <textarea
        name="msg"
        value={input.msg}
        onChange={handleChange}
      ></textarea>
      <br />
      <br />
      <button type="submit">Send</button>
    </form>
  );
}

Explanation:

  1. Using fetch API:

    • fetch is used to send a POST request to the specified URL.
    • method specifies the HTTP method, in this case, 'POST'.
    • headers specifies the content type as JSON, which is necessary for sending JSON data.
    • body contains the JSON stringified input data.

  2. Error Handling:

    • Checks if the response is not OK and throws an error if it isn’t.
    • Logs errors to the console and can be used to show error messages to the user.

  3. Handling Response:

    • Waits for the response to be converted to JSON and logs the result.
    • You can handle the result as needed, such as showing a success message or resetting the form.

This approach will work similarly to axios for sending HTTP requests but uses the built-in fetch API, which is part of modern JavaScript.
